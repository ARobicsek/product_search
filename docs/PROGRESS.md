**Finding worth remembering:** A bug surfaced on one product can have generic roots if it shares a code path with every other product. The "$3.44 cheapest subscription" defect read like a magazine-only extraction bug, but the actual code (`_calculate_total`) had been silently RAM-only for every non-RAM product in the repo for months — magazines just made the consequence numerically dramatic. Default columns inherited the same RAM-shape. Whenever a defect is reported "on product X," check whether the suspect function/default has a RAM-vs-non-RAM seam — if yes, the bug almost certainly extends to every non-RAM product, and the fix should generalize, not patch. Pre-emptive rule: any function gated on `attrs.capacity_gb` is a candidate for the same audit (see also ADR-085 D2 — the per-source-attribution bug had the same generic-vs-RAM seam).
