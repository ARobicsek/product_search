- **Most recent work:** 2026-05-28 **Phase 29 ADR-114: server-streamed draft updates keep the onboard preview in sync during tool-use loops.**
  - **ADR-114:** A live 8-vendor DJI Neo 2 re-onboard left the preview pane stuck on `{}` despite the LLM saying it added all 8 vendors. Root cause: `findLatestDraft` reads `<draft>` blocks from assistant text, but Anthropic stops each message at the first `tool_use` — so during a long probe-then-fix-detail-URLs loop the LLM never emits a closing `</draft>`, the client falls back to turn-1's empty stub, and the 50s force-finalize then often returns an incomplete draft. One new guard test (29 total); web `tsc`/`eslint`/`test:guards 29/29`/`test:parity 6/6`/`next build` clean.
