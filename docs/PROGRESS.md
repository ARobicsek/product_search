## Current state — 2026-05-26 ADR-098 implemented (supermicro-zero-results)

**Deliverables:** ADR-098 in DECISIONS.md (ACCEPTED). Five fixes for the defect class where bad onboarder URLs sail through and get reported with misleading messages.

**What shipped (commit `725f590`):**
- **Fix #1 — relevance-aware search probe.** `probe-url.ts` returns `relevanceHits` (product anchors whose text matches the target's distinctive tokens). New `countRelevanceHits()` + `distinctiveTokens()`. `route.ts` exposes `target_name` in the `probe_url` tool schema so the LLM can pass it. Prompt documents advisory "many anchors, 0 relevance = mis-scoped URL" interpretation.
- **Fix #2 — Newegg search pattern + `N=` trap.** Prompt now includes `newegg.com/p/pl?d=<keywords>` in known patterns and warns that `N=<digits>` category-node params silently override keywords.
- **Fix #3 — low-body floor.** `THIN_BODY_CEILING = 5_000` in `source_reasons.py`. A sub-5K body with 0 candidates → `TRANSIENT` ("small body — likely a stub or blocked page; re-check the URL"), not `EMPTY_PAGE`.
- **Fix #4 — dominant-rejection messaging.** `cli.py` computes `dominant_rejection` from `ai_filter.LAST_RUN_LOG` per source (≥50% `relevance_check` → `"relevance_check"`). Classifier says "mis-scoped" instead of "loosen your filter".
- **Fix #5 — no-guessed-slug rule.** Prompt now says "NEVER construct a detail URL by guessing a slug pattern."
- All prompt changes synced via `sync-prompt.js`.
- Green: worker pytest **16/16** (4 new); web `tsc --noEmit` 0 errors; `test:guards` 15/15 (4 new); `test:parity` 4/4.
